- **Objective:** Fix backend `.env` loading.
- **Action:** Fixed the bug where the backend was returning `undefined` for `MONGODB_URI`. Updated `server/src/config/env.js` to explicitly tell `dotenv` to look up one directory (`path: '../.env'`), allowing it to successfully read the single root coordinate file we set up.

- **Objective:** Improve frontend validation error UX.
- **Action:** Modified the `handleValidationErrors` middleware in `server/src/modules/auth/auth.validation.js`. We switched the master response `message` from the generic `"Validation failed"` string to the actual error message generated by the specific validation rule (e.g., `"Please use college email only (@bbdniit.ac.in)"`). This allows the frontend to show the exact reason why registration failed immediately in its UI.

- **Objective:** Build robust virtual practical and code evaluation module.
- **Action:** Built a full vertical slice for the interactive 'Practicals' request. Using `@monaco-editor/react`, Node.js native `vm` module runtime isolation for code execution, schema-driven MongoDB submissions, and responsive React hooks, we now have a working sandbox tab logic allowing teachers to provision tests and students to execute live code against expected outputs.
